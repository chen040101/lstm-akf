# 开发文档

## 1. 当前目标

项目当前聚焦一件事：

- 从 YOLO 检测结果中取出当前最好打击的装甲板中心点；
- 用 AKF 做去噪、短时补位和未来基线外推；
- 用 LSTM-AKF 预测未来 `1~15` 帧归一化中心坐标；
- 连续漏检超过 `5` 帧时停止当前轨迹预测。

## 2. 当前实现概览

### 数据流

1. `YOLO -> (cx, cy)`：统一转换为归一化中心点；
2. `AKF`：维护 `[cx, cy, vx, vy, ax, ay]` 状态，短时漏检时补位；
3. `数据集构建`：从连续轨迹段切出 `input/target` JSON；
4. `LSTM-AKF`：读取历史 `xy`，结合 AKF 基线做门控融合预测；
5. `训练`：记录 `loss/ADE/FDE`，并支持早停；
6. `模型对比`：固定对比多种基线，`LSTM-AKF` 只加载外部 checkpoint。

### 当前默认值

- 历史长度：最少 `5` 帧，最多 `15` 帧；
- 预测范围：未来 `1~15` 帧；
- 目标时间尺度：`40 FPS`；
- 数据集划分：默认 `segment` 级划分；
- 主训练早停：默认 `patience=20`，`min_delta=0.0`。

## 3. 核心代码位置

- `scripts/build_dataset.py`：数据集构建入口；
- `scripts/train.py`：主模型训练入口；
- `scripts/compare_models.py`：固定模型对比入口；
- `lstm_akf/datasets/builder.py`：轨迹切样、train/val 划分、manifest 导出；
- `lstm_akf/models/lstm_akf.py`：门控融合版 LSTM-AKF；
- `lstm_akf/training/runner.py`：训练、保存 checkpoint、早停；
- `lstm_akf/training/comparison.py`：对比流程与结果汇总。

## 4. 当前实现边界

- 神经网络输入仍然只有 `xy`；
- 速度、加速度由 AKF 内部估计和外推，不直接写入 JSON 样本；
- 数据集不会跨轨迹段拼接；
- `LSTM-AKF` 对比模型不会自动训练，缺 checkpoint 就跳过。
