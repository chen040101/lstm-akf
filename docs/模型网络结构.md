## MLP: 64→128→64→30 的含义

这里的 `MLP: 64→128→64→30` 指的是 `Residual Head` 和 `Direct Head`
内部全连接层的维度变化，不是时间步数量的变化。

### 1. 输入 64 维

LSTM Encoder 会把历史中心点序列：

```text
history_xy: [B, 15, 2]
```

编码成一个历史运动特征：

```text
history_embedding: [B, 64]
```

所以两个预测头的 MLP 输入维度都是 `64`。

### 2. 中间升维到 128，再降到 64

代码里的预测头结构是：

```text
Linear(64, 128)
ReLU
Dropout
Linear(128, 64)
ReLU
Linear(64, 30)
```

含义是：

```text
64 维历史特征
→ 128 维隐藏特征
→ 64 维隐藏特征
→ 30 维输出
```

其中 `128` 用来增强非线性表达能力，后面再压回 `64`，最后映射到预测结果。

### 3. 为什么最后是 30

模型要一次预测未来 `15` 帧，每一帧有中心点坐标 `(x, y)` 两个值：

```text
future_steps = 15
每帧输出维度 = 2
15 × 2 = 30
```

所以 MLP 最后一层输出：

```text
[B, 30]
```

随后代码会 reshape 成：

```text
[B, 15, 2]
```

也就是未来 1~15 帧的中心点序列。

### 4. 两个预测头的区别

两个 Head 的 MLP 结构相同，但学习目标不同：

```text
Residual Head:
history_embedding [B,64]
→ pred_delta_xy [B,15,2]
→ residual_pred_xy = baseline_xy + pred_delta_xy

Direct Head:
history_embedding [B,64]
→ direct_offset_xy [B,15,2]
→ direct_pred_xy = last_xy + direct_offset_xy
```

也就是说：

- `Residual Head` 学的是对 AKF 基线轨迹的修正量；
- `Direct Head` 学的是从最后一个历史点直接外推的偏移量。

### 5. Gate Head 的维度补充

门控分支不是 `64→128→64→30`，而是：

```text
68→64→1
```

原因是门控输入由三部分拼接：

```text
repeat(history_embedding): 64 维
residual_pred_xy: 2 维
direct_pred_xy: 2 维
```

因此每个未来时间步的 gate 输入维度是：

```text
64 + 2 + 2 = 68
```

Gate Head 输出每个未来时间步的一个融合权重：

```text
fusion_gate: [B,15,1]
```

最终融合公式为：

```text
pred_xy = gate * residual_pred_xy + (1 - gate) * direct_pred_xy
```
