# 模型结构设计

## 1. 当前模型概览

主模型不是“固定相加”了，而是门控融合的 `LSTM-AKF`：

```text
历史 xy -> LSTM 编码
        -> 残差分支
        -> 直推分支
AKF 基线 --------->
三者通过 gate 融合，得到未来 15 帧预测
```

网络实现位于：`lstm_akf/models/lstm_akf.py`。

## 2. 输入与输出

### 输入

- `history_xy`：`[B, 15, 2]`
- `history_len`：`[B]`
- `baseline_xy`：`[B, 15, 2]`

其中：

- 网络输入特征仍然只有 `xy`；
- `history_xy` 是左侧 padding 后的历史序列；
- `baseline_xy` 由 AKF 根据历史序列外推得到。

### 输出

- `pred_xy`：最终未来 `1~15` 帧预测；
- `fusion_gate`：形状为 `[B, 15, 1]` 的门控权重；
- 同时保留 `residual_pred_xy`、`direct_pred_xy` 便于训练和分析。

## 3. AKF 层职责

AKF 负责两件事：

- 维护 `[cx, cy, vx, vy, ax, ay]` 状态；
- 根据历史序列生成未来 `1~15` 帧基线轨迹。

也就是说，速度和加速度已经进入系统，但它们来自 AKF 状态估计，不是额外写入 JSON 的神经网络输入特征。

## 4. 门控融合方式

### 分支定义

- 残差分支：`residual_pred = baseline_xy + pred_delta_xy`
- 直推分支：`direct_pred = last_xy + direct_offset_xy`
- 门控分支：`gate = sigmoid(g(history_embedding, residual_pred, direct_pred))`

### 最终输出

```text
pred = gate * residual_pred + (1 - gate) * direct_pred
```

含义：

- 当 AKF 基线更可靠时，模型可以把权重更多给残差分支；
- 当历史模式更像直接时序外推时，模型可以更多依赖直推分支；
- 融合权重是逐时间步学习出来的，不再是固定相加。

## 5. `lstm_only` 分支

当 `use_baseline=False` 时：

- 不使用 AKF 基线融合；
- `pred_xy = direct_pred_xy`；
- 该配置主要用于基线对比，不是主模型。

## 6. 训练目标

当前训练损失由三部分组成：

- 主损失：`loss(pred_xy, target_xy)`
- 残差辅助损失：`loss(residual_pred_xy, target_xy)`
- 直推辅助损失：`loss(direct_pred_xy, target_xy)`

主训练默认：

- `residual_aux_weight = 0.2`
- `direct_aux_weight = 0.2`

基础损失是 `SmoothL1Loss`，训练和验证都记录：

- `loss`
- `ADE`
- `FDE`

## 7. 训练控制

主训练在 `lstm_akf/training/runner.py` 中支持早停：

- 监控验证集 `loss`；
- 如果没有验证集，则退化为监控训练集 `loss`；
- 默认 `--early-stopping-patience 20`。

这部分属于训练策略，但和当前模型使用方式强相关，因此一并记录在这里。
