# 模型对比

## 1. 固定对比模型

当前固定对比以下模型：

- `LSTM-AKF`
- `AKF-only`
- `LSTM-only`
- `ANN`
- `SVR`
- `AR`
- `DT`
- `KNN`

## 2. 当前行为约定

### `LSTM-AKF`

- 对比脚本只尝试加载外部 checkpoint；
- 期望路径：`Baseline Models/lstm_akf/checkpoints/best.pt`；
- 会自动创建目录，但不会自动训练；
- 如果该文件不存在，就直接跳过，并把原因写入结果文件。

### `AKF-only`

- 不训练；
- 直接使用数据加载阶段生成的 AKF 基线作为预测结果。

### `LSTM-only`

- 会在 `Baseline Models/lstm_only/` 下自动训练或复用已有 `best.pt`；
- 若数据集或配置变化，会重新训练。

### 传统机器学习模型

- `ANN / SVR / AR / DT / KNN` 会自动训练或复用 `artifacts/model.pkl`；
- 如果没有 `scikit-learn`，对应模型会被跳过。

## 3. 默认评价范围

默认比较两类范围：

- 单帧：`--point-frames 8 15`
- 连续区间：`--sequence-horizons 8 15`

统一输出指标：

- `DE`
- `MSE`
- `RMSE`
- `MAE`
- `Hit Rate`
- `Inference Time`

其中：

- `Hit Rate` 按 `DE <= hit_threshold` 统计；
- 当前脚本默认 `hit_threshold = 0.02`；
- 如需调整，可在对比命令中增加 `--hit-threshold`。

## 4. 输出位置

对比结果统一写入：`Baseline Models/`

其中常用内容包括：

- `Baseline Models/<model_name>/`：各模型自身产物；
- `Baseline Models/comparison/summary_results.json`：汇总结果；
- `Baseline Models/comparison/detailed_results.json`：详细结果；
- `Baseline Models/comparison/comparison.csv`：表格结果；
- `Baseline Models/comparison/summaries/*.md`：按范围生成的简要说明；
- `Baseline Models/comparison/*.png`：指标图。

## 5. 常用命令

```bash
conda activate yolo
python scripts/compare_models.py --device cuda --point-frames 8 15 --sequence-horizons 8 15
```

## 6. 使用建议

如果你想把自己的主模型权重纳入固定对比：

1. 先把 `best.pt` 放到 `Baseline Models/lstm_akf/checkpoints/`；
2. 再运行 `compare_models.py`；
3. 若没有放置该文件，对比仍会继续，只是不会包含 `LSTM-AKF` 这一项。
