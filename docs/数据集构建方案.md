# 数据集构建方案

## 1. 默认目录

项目默认使用以下目录：

- 视频输入：`data/video/`
- YOLO 权重：`weights/yolo/best.pt`
- 训练集输出：`data/dataset/train/`
- 验证集输出：`data/dataset/val/`
- 划分清单：`data/dataset/split_manifest.json`

## 2. 样本格式

每个 JSON 只保留两部分：

```json
{
  "input": [[x1, y1], [x2, y2], [x3, y3]],
  "target": [[x4, y4], [x5, y5], [x6, y6]],
  "meta": {
    "video_name": "test2.mp4",
    "segment_index": 11,
    "history_frame_indices": [100, 101, 102],
    "target_frame_indices": [103, 104, 105],
    "history_timestamps": [2.500, 2.525, 2.550],
    "target_timestamps": [2.575, 2.600, 2.625]
  }
}
```

说明：

- `input`：过去 `5~15` 帧中心点序列；
- `target`：未来 `1~15` 帧中心点序列；
- `meta`：用于把样本映射回真实视频帧，训练时会被忽略；
- 样本训练只使用 `xy`，AKF 基线在加载数据时根据历史序列重新生成。

## 3. 构建流程

1. 对 `data/video/` 下视频逐帧做 YOLO 检测；
2. 每帧只保留当前最好打击的装甲板中心点；
3. 用 AKF 平滑轨迹，并在短时漏检时补位；
4. 连续漏检超过 `5` 帧则切成新轨迹段；
5. 轨迹按 `40 FPS` 重采样；
6. 对每段轨迹做滑窗切样；
7. 按 train/val 输出 JSON，并记录划分 manifest。

## 4. 划分规则

支持三种划分方式：

- `sample`：按样本随机划分；
- `segment`：按连续轨迹段划分；
- `video`：按整段视频划分。

当前默认值是：

```text
--split-mode segment
```

这样可以避免同一轨迹段同时出现在训练集和验证集，通常比 `sample` 更稳妥。

`split_manifest.json` 会记录：

- 划分模式；
- 随机种子；
- train/val 的样本数；
- 每个 group 属于 train 还是 val。

## 5. 常用命令

在项目根目录下执行：

```bash
conda activate yolo
python scripts/build_dataset.py --val-ratio 0.2 --split-mode segment
```

如果已经有轨迹段文件，也可以直接切样：

```bash
python scripts/build_dataset.py --tracks-input data/tracks/segments.pt --val-ratio 0.2 --split-mode segment
```

## 6. 备注

- `--val-ratio <= 0` 时只导出训练集；
- 输出目录中的旧 JSON 会先清理再重建；
- 若后续需要更强隔离，可把 `split_mode` 改成 `video`。
